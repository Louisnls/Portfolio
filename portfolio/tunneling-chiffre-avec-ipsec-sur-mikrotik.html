Introduction

Dans ce labo, nous allons réaliser un tunnel Eoip entre hôte distant, avec DHCP qui passe par ce tunnel :

Voici notre topologie. Le but est, que chaque routeur soit à la fois le client et le serveur DHCP de chacun des autres routeurs. Les configurations DHCP passant par notre tunnel, le tout chiffré avec IpSec.

Mise en place du tunnel Eoip avec IpSec

Disons que nous sommes sur le "Router-Louis". Nous allons d'abord mettre en place notre serveur DHCP, quand "Router-Cheridanh" sera notre client : 

/interface eoip add name="eoipservercheridanh" tunnel-id=2 remote-address=10.190.0.85 disabled=no 
/interface eoip set allow-fast-path=no 
/intercace eoip set ipsec-secret=iticparis.com

Nous renseignons ici comme remote-addresse son IP WAN, avec comme tunnel ID 2, et chiffrer avec IpSec par le mot de passe "iticparis.com

Chaque tunnel se doit d'avoir un unique ID. Le client DHCP du "Router-Cheridanh" devra donc avoir lui aussi comme tunnel id le 2 pour recevoir notre configuration DHCP.

Configurons maintenant notre routeur quand nous sommes le client DCHP du "Router-Cheridanh" :

/interface eoip add name="client-cheridanh" tunnel-id=0 remote-address=10.190.0.85 disabled=no 
/interface eoip set allow-fast-path=no 
/interface eoip set ipsec-secret=iticparis.com

Pareil que lors de la configuration pour le serveur DHCP, nous renseignons l'adresse IP WAN du "Router-Cheridanh" et choisissons un autre tunnel ID.

Une fois les 3 routeurs configurés, nous devons nous retrouver avec 4 interfaces, car je peux être à la fois serveur DHCP de "Router-Victor" et "Router-Cheridanh", et client du DHCP de "Router-Cheridanh" et/ou de "Router-Victor"

Configuration des interfaces et du serveur DHCP

Maintenant, nous devons bridger nos deux interfaces Eoip serveur ensemble, et l'interface où je suis le client de Cheridanh avec les ports de mon router où je veux que mes posts récupèrent son DHCP, de même avec l'interface où je suis le client de Victor. 

Quand je suis serveur :

/interface bridge add name=bridgeserver
/interface bridge port add bridge=bridgeserver interface=eoipservercheridanh
/interface bridge port add bridge=bridgeserver interface=eoipservervictor
/interface bridge port add bridge=bridgeserver interface=ether4
/interface bridge port add bridge=bridgeserver interface=ether5

Quand je suis client de Cheridanh :

/interface bridge add name=bridgeclientcheridanh
/interface bridge port add bridge=bridgeclientcheridanh interface=client-cheridanh
/interface bridge port add bridge=bridgeclientcheridanh interface=ether2
/interface bridge port add bridge=bridgeclientcheridanh interface=ether3

Quand je suis client de Victor :

/interface bridge add name=bridgeclientvictor
/interface bridge port add bridge=bridgeclientvictor interface=clientvictor
/interface bridge port add bridge=bridgeclientvictor interface=ether6
/interface bridge port add bridge=bridgeclientvictor interface=ether7

 

Nous devons nous retrouver avec quelque chose comme ceci :

Nous devons maintenant ajouter une addresse ip sur nos 3 bridges créés. Les addresses ip doivent être dans le même réseau avec nos hôtes distant.

/ip address add address=192.168.100.1/24 interface=bridgeserver
/ip address add address=192.168.110.2/24 interface=client-cheridanh
/ip address add address=192.168.120.2/24 interface=bclientvictor

Pour finir, on met un DHCP sur notre interface "bridgeserver" en faisant :

/ip dhcp-server setup

dhcp server interface: bridgeserver 

dhcp address space: 192.168.100.0/24 

gateway for dhcp network: 192.168.100.1 

dhcp relay: 192.168.0.1 

addresses to give out: 192.168.100.2-192.168.100.254 

dns servers: 192.168.100.1 

lease time: 1d

Tout est maintenant prêt, nos posts clients peuvent recevoir leur addresse ip ainsi que pour les posts clients des hôtes distant.
